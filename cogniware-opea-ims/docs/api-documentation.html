Authentication

All API requests (except /health and /status) require authentication using Bearer tokens.

POST /auth/login
Authenticate with username and password to receive an access token.

Request Body

{ "username": "your_username", "password": "your_password" }

Response

{ "success": true, "data": { "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...", "expires_in": 3600, "refresh_token": "refresh_token_here" } }

Using the Token

Model Management

GET /models
List all loaded models

Response

{ "models": [ { "model_id": "llama-7b", "device_id": 0, "status": "loaded", "memory_usage_mb": 14336 } ] }
POST /models
Load a new model onto specified GPU. Expected load time: ~3 seconds (15x faster than traditional systems).

Request Body

{ "model_id": "llama-7b", "model_path": "/models/llama-7b.bin", "device_id": 0, "precision": "fp16", "max_batch_size": 32 }
DELETE /models/{model_id}
Unload a model from GPU memory

Request Body

Multi-LLM Orchestration

🔥 60,000 Tokens/Second

4 models running in parallel - 120x faster than traditional single-model systems

POST /orchestration/parallel
Run inference on 4 models simultaneously. Returns results from all models.

Request Body

{ "prompt": "Summarize quantum computing in one sentence", "model_ids": [ "llama-7b", "llama-13b", "gpt-7b", "mistral-7b" ], "max_tokens": 50 }

Response

{ "results": { "llama-7b": { "text": "Quantum computing uses quantum...", "time_ms": 9.2 }, "llama-13b": { /* ... */ }, "gpt-7b": { /* ... */ }, "mistral-7b": { /* ... */ } }, "total_time_ms": 10.5 }
POST /orchestration/consensus
Get consensus answer from multiple models with confidence scoring

Request Body

{ "prompt": "Is climate change real?", "model_ids": ["llama-7b", "gpt-7b", "mistral-7b"], "consensus_threshold": 0.75 }

Response

{ "consensus_answer": "Yes, climate change is real...", "confidence": 0.95, "agreement_percentage": 100, "individual_responses": [/* ... */] }

Model Context Protocol (MCP) Tools

Cogniware Core features complete MCP integration with 10 subsystems providing full platform automation and control capabilities.

MCP Tool Execution Endpoint

POST /mcp/tools/execute
Execute any MCP tool with parameters

Request Format

{ "tool": "tool_name", "parameters": { "param1": "value1", "param2": "value2" } }

Available MCP Subsystems

Subsystem Tools Available Description
Filesystem read_file, write_file, list_directory, search_files, delete_file, create_directory Complete file and directory operations
Internet http_get, http_post, websocket_connect, api_call, scrape_website HTTP, HTTPS, WebSocket, API integration
Database db_connect, db_query, db_select, db_insert, db_update, db_delete SQL (PostgreSQL, MySQL, SQLite) and NoSQL (MongoDB, Redis)
Applications launch_process, execute_command, kill_process, list_processes, open_application Process and application lifecycle management
System Services get_system_metrics, log_message, query_logs, start_service, stop_service System monitoring, logging, and service control
Security authenticate, validate_token, check_permission, create_user, hash_password Authentication, authorization, and security operations
Resources get_memory_info, get_cpu_info, get_gpu_info, allocate_resource, get_resource_usage Resource monitoring and allocation
Tool Registry list_tools, search_tools, get_tool_info, execute_tool Tool discovery and management

Read File

{ "tool": "read_file", "parameters": { "path": "/data/document.txt" } } // Response { "success": true, "result": "File contents here..." }
